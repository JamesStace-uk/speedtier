Sources, Licenses, and Method

Data Source

Primary Pokemon data source: PokeAPI.

PokeAPI repositories referenced for data and API service: PokeAPI/pokeapi and PokeAPI/api-data. Both repositories currently report BSD 3-Clause licensing.

Sprite and Asset Sources

Pokemon sprites in this tool are sourced through PokeAPI sprite metadata, with this build preferring Pokemon Home, Scarlet/Violet, Emerald, Brilliant Diamond / Shining Pearl, then PokeAPI front_default, before falling back to a locally generated placeholder.

Upstream sprite repository: PokeAPI/sprites. That repository uses its own LICENCE.txt and documents mixed sprite provenance, including some community-created sprite sets.

Additional community sprite reference: smogon/sprites. The repository code is MIT-licensed, while its README states the sprites themselves are property of Nintendo / Game Freak / The Pokemon Company, with some community-created sprite licensing still being determined.

Favicon and Fallback Assets

Favicon asset: the Poke Ball item sprite from PokeAPI/sprites.

Fallback sprite asset: generated locally by this project as a transparent placeholder image.

Code Method

Primary external data source: PokeAPI.

Development method for this project: vibecoding / AI-assisted iterative development.
